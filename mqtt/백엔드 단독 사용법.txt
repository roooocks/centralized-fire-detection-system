MQTT broker는 MQTT Mosquitto를 사용합니다. 오라클 사에서 관리하는 오픈소스 broker입니다. borker의 동작 방식은 사실상 중계기라고 보시면 될 것 같습니다. A가 통신을 보내면 B가 그걸 받고, 그 반대 역시 가능합니다.


broker를 윈도우에서 받을 경우 mosquitto-2.1.2-install-windows-x86.exe 형식의 파일이 다운로드 폴더에 들어가게 됩니다. (x86은 뭘 받느냐에 따라 달라짐)
해당 파일을 더블 클릭하여 설치하시면 됩니다. (설치 옵션은 건드릴게 없음)


설치가 끝난 borker를 그대로 사용하면 안되며, 아래와 같은 설정을 borker를 설치하여 생긴 폴더 내부의 mosquitto.conf 파일에 메모장으로 접속하여 적용할 필요가 있습니다.
    1. 외부(게이트웨이, 다른 서버 등)와 같이 쓸 경우
        - 대분자 소문자 구별해서 "# Listeners"를 찾습니다.
        - 살짝 내리면 "# listener"을 찾을 수 있습니다.
        - #과 띄어쓰기를 지워서 라인에 딱 붙인 상태로 "listener 1883 192.168.0.10"을 작성합니다.
        - 대문자 소문자 구별해서 "# Security"를 찾습니다.
        - 살짝 내리면 "# allow_anonymous false"를 찾을 수 있습니다.
        - #과 띄어쓰기를 지워서 라인에 딱 붙인 상태로 "allow_anonymouse true"를 작성합니다.
    2. 백엔드에서 테스트를 위해 단독으로 쓸 경우
        - 대분자 소문자 구별해서 "# Listeners"를 찾습니다.
        - 살짝 내리면 "# listener"을 찾을 수 있습니다.
        - #과 띄어쓰기를 지워서 라인에 딱 붙인 상태로 "listener 1883"을 작성합니다.
        - 대문자 소문자 구별해서 "# Security"를 찾습니다.
        - 살짝 내리면 "# allow_anonymous false"를 찾을 수 있습니다.
        - #과 띄어쓰기를 지워서 라인에 딱 붙인 상태로 "allow_anonymouse true"를 작성합니다.


broker를 바꾼 옵션과 함께 실행하기 위해서 broker가 설치된 폴더로 들어가 아래와 같은 명령어를 사용합니다. (윈도우 cmd 기준 / 이미지 폴더의 image01)
.\mosquitto.exe -c mosquitto.conf -v

위 명령어를 사용해서 이미지처럼 나온다면 중계기는 준비가 끝났습니다. 이제 클라이언트단에서 코드를 작성해야 합니다.
이때 사용하는 MQTT는 MQTT client로, Express라면 mqtt.js를 Python이라면 paho-mqtt라는 패키지를 사용하게 됩니다. 본인이 어떤 언어와 프레임워크를 사용하는지에 따라 다르니 꼭 AI에게 물어봅시다.



이제 MQTT 통신 구조에 대해서 자세하게 설명하고 넘어가겠습니다.
MQTT의 통신 구조는 "A <> (전송) <> broker <> (전송) <> B" 입니다. topic과 payload를 합치는 방식으로 데이터 만들고 publish(데이터 전송), subscribe(구독) 방식으로 통신을 진행합니다.
A에서 B로 보낸다면 A가 topic과 payload를 만들어서 publish로 데이터를 전송합니다.
B는 subscribe를 통해 토픽들을 구독해놓고 무한 대기합니다. 만약 토픽이 온다면 해당 토픽의 이벤트에 따른 동작을 작동시키면 됩니다. 예제 코드는 "백엔드 단독 사용법의 명령어 리뉴얼.txt" 텍스트 파일 확인 부탁드립니다.



제가 최대한 알고 있고, 알아낸 방식으로 작성했습니다. 이 이상은 저도 이제 감지기/게이트웨이를 작업해야하고, 테스트를 빠르게 해야해서 AI 쓴거라 자세하게 적는건 무리였습니다 ㅠ

소켓을 평범하게 쓴다면 건물 A, B 2개가 저희 상품을 사용한다는 가정 하에, A에서 화재 감지 시 A와 B 모두에게 알림을 보내버리는 대참사가 일어납니다.
때문에 소켓을 room 방식으로  이용해서 특정 room에 메시지를 보내는 방식을 채택해야 합니다.
io.on('connection', (socket) => {
  // room 참가
  socket.join('company_123');

  // 특정 room에 메시지 보내기
  io.to('company_123').emit('status', data);

  // 나가기
  socket.leave('company_123');
});


하지만 저희는 건물이 하나뿐인지라 굳이 이렇게 하지 않아도 됩니다.
이유는 간단한데, 재부팅 시 소켓 정보가 다 날아가서 접속을 다시 하려면 DB를 불러와야 하고, 현재 UI 디자인이 이쪽에 맞게 되어있지 않아서 그렇습니다.
